# Hidayah Backend Service (Dua Backend)

> Is just a bootstrap project for Hidayah, don't expect much from it. will do things properly later

This is the backend service for the **Hidayah** application. It provides serverless REST APIs to generate contextual Islamic Duas using Google's Gemini AI and synthesize Arabic text to speech using Google Cloud Platform (GCP) Text-to-Speech (TTS).

The project is built entirely in **Rust** optimized for **AWS Lambda** and deployed using **AWS Serverless Application Model (SAM)**.

## Features

- ⚡ **Serverless & Fast**: Built with Rust & `cargo-lambda` for extremely low cold-starts and low memory consumption.
- 📡 **Server-Sent Events (SSE)**: Streams generated Duas directly to the client as they are generated by Gemini AI.
- 🗣️ **Text-to-Speech**: Synthesizes high-quality Arabic audio from generated Duas using GCP TTS.
- 🔐 **Secure Configuration**: Retrieves sensitive credentials (like the GCP API key) dynamically from AWS Systems Manager (SSM) Parameter Store.

## Architecture Overview

The backend is composed of an **AWS API Gateway** pointing to two primary **Lambda functions**:

1. **`GenerateDua`** (`POST /dua/generate`)
   - Uses the `google-ai-rs` crate to call the Gemini API.
   - Responds using `RESPONSE_STREAM` InvokeMode to stream words back to the client via SSE (`text/event-stream`).
2. **`GenerateAudio`** (`POST /dua/audio`)
   - Validates that the input text is Arabic.
   - Calls GCP Text-to-Speech API to synthesize the audio payload (supports `male` and `female` genders).

## Prerequisites

Before building or deploying the service locally, ensure the following are installed:
- [Rust](https://www.rust-lang.org/tools/install)
- [Cargo Lambda](https://www.cargo-lambda.info/) for compiling lambdas.
- [AWS CLI](https://aws.amazon.com/cli/) & configured credentials.
- [AWS SAM CLI](https://docs.aws.amazon.com/serverless-application-model/latest/developerguide/install-sam-cli.html) for deployment.

## Environment Variables & Configuration

The lambdas depend on a Google (GCP) API Key to access both Gemini AI and GCP TTS. This key is fetched securely at runtime from **AWS Systems Manager (SSM) Parameter Store**.

Your AWS environment should have a `SecureString` parameter at the following path:
```
/hidayah/{stage}/gcp-api-key
```
*(where `{stage}` is the target environment, e.g., `dev` or `prod`)*.

## API Endpoints

### 1. Generate Dua
**POST** `/{stage}/dua/generate`

Generates a contextual Dua based on a user prompt and streams the response back via Server-Sent Events (SSE).

**Request (JSON)**
```json
{
  "prompt": "For protection while traveling"
}
```

**Response (SSE Stream)**
The response will stream multiple JSON chunks:
```text
data: {"type": "chunk", "text": "Allahumma "}
data: {"type": "chunk", "text": "innee "}
...
data: {"type": "done"}
```

### 2. Generate Audio
**POST** `/{stage}/dua/audio`

Synthesizes the given Arabic text into an audio payload.

**Request (JSON)**
```json
{
  "text": "اللَّهُمَّ إِنِّي أَسْأَلُكَ",
  "gender": "male" // optional, defaults to "male". Accepts "male" or "female".
}
```

**Response (JSON)**
```json
{
  "statusCode": 200,
  "message": "Audio generated successfully",
  "data": {
    "audioContent": "<base64_encoded_audio>"
  }
}
```

## Build & Development

The project includes an automated `Makefile` for streamlined development and building.

### Quick Start
To build the lambdas for AWS:
```bash
make build
# or
make build-all
```
This command parses the `Cargo.toml`, discovers the binaries (`GenerateDua`, `GenerateAudio`), builds them using `cargo lambda`, and automatically copies the execution bootstraps into `.aws-sam/build/` to prepare for AWS SAM deployment.

### Useful Make Commands
- `make check` - Runs standard cargo checks across all features.
- `make test` - Runs integration and unit tests.
- `make clean` - Cleans target bundles and the SAM build directory.
- `make info` - Shows current environment information, Rust versions, and auto-discovered binaries.
- `make list` - Lists the binaries available and checks their specific feature requirements.
- `make sizes` - Displays the output bundle sizes.
- `make dev` - Complete local workflow (validation, check, test, build, and sizes).

## Deployment

Deploy using AWS SAM after building:
```bash
make build
sam deploy --guided
```
Follow the SAM CLI prompts to configure stack name, region, and parameter overrides (like `StageName`).
